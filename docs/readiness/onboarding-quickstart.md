# Readiness Onboarding Quickstart

Use this guide to onboard new contributors into the readiness system quickly.

## First 30 minutes

1. Read:
   - `docs/readiness/master-toc.md`
   - `docs/readiness/operator-cockpit.md`
   - `docs/readiness/role-directory.md`
2. Confirm your assigned role and backup owner.
3. Review today’s top blockers in `operator-cockpit.md`.
4. Run baseline command:
   - `pnpm claims:check`

## First day

1. Review current cycle state:
   - `30-day-master.md`
   - `go-no-go-decision.md`
   - `risk-register.md`
2. Check active actions in `owner-action-register.md`.
3. Add at least one evidence-linked update to `evidence-log.md`.
4. Attend daily blocker sync and weekly review if set to `ready`.

## First week

1. Deliver one owned action end-to-end with evidence.
2. Update one weekly artifact (`weekly-review-*`, scorecard, or evidence log).
3. Participate in at least one readiness gate discussion.
4. Confirm your handoff/update style follows templates.

## Common mistakes to avoid

- Updating status without linking evidence.
- Leaving ownership fields vague.
- Making exception/risk changes without expiry dates.
- Skipping `claims:check` before substantive edits.
