# UX Regression Checklist

Run this checklist before weekly gate review and RC promotion.

## Core UX checks

- [ ] Primary navigation works on supported surfaces.
- [ ] Error/empty/loading states are visible and understandable.
- [ ] Forms validate correctly and show actionable errors.
- [ ] Critical CTA buttons are reachable and functional.

## Accessibility checks

- [ ] Keyboard reachability on key paths (web).
- [ ] Focus states visible for interactive controls.
- [ ] Labels and key copy are clear for assistive use.

## Visual consistency checks

- [ ] No major layout breaks on common viewport sizes.
- [ ] Core typography/colors remain consistent in key flows.
- [ ] No blocking visual regressions in trip, listing, or profile journeys.

## Evidence

- [ ] Results linked in rehearsal report or evidence log.
