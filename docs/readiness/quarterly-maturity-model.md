# Quarterly Readiness Maturity Model

Use this model to assess and improve readiness capabilities each quarter.

## Maturity levels

| Level | Name | Characteristics |
| --- | --- | --- |
| 1 | Ad hoc | Inconsistent process, evidence gaps, unclear ownership |
| 2 | Managed | Core routines exist, but quality varies by owner |
| 3 | Defined | Standardized workflows, documented gates, reliable evidence |
| 4 | Measured | KPI-driven control loops with predictable outcomes |
| 5 | Optimizing | Continuous improvement with fast feedback and low incident debt |

## Capability domains

| Domain | Description |
| --- | --- |
| Governance | Decision cadence, ownership clarity, escalation behavior |
| Quality gates | Command checks, manual checks, release controls |
| Evidence integrity | Traceable evidence quality and freshness |
| Security and compliance | Control coverage, sign-off discipline, exception handling |
| Incident readiness | Classification, response speed, drill effectiveness |
| Post-launch learning | Retrospectives, lessons-to-actions closure |

## Scoring method

- Score each domain 1-5.
- Compute average maturity score.
- Identify lowest two domains for next-quarter improvement focus.

## Improvement rule

- No domain should remain at level 1 for two consecutive quarters.
- Any domain below level 3 requires named owner and roadmap actions.
