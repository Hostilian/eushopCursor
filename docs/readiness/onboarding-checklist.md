# Readiness Onboarding Checklist

Use this checklist for each new teammate joining readiness execution.

## Setup

- [ ] Role assignment confirmed in `role-directory.md`.
- [ ] Backup owner identified.
- [ ] Communication channel confirmed.

## Context

- [ ] Read `master-toc.md`.
- [ ] Read `operator-cockpit.md`.
- [ ] Read `owner-action-register.md`.
- [ ] Read `go-no-go-decision.md`.

## Execution readiness

- [ ] Ran `pnpm claims:check`.
- [ ] Knows where to log evidence (`evidence-log.md`).
- [ ] Knows where to log decisions (`30-day-master.md`).
- [ ] Knows where to log risks (`risk-register.md`).

## First contribution quality bar

- [ ] Evidence attached to status update.
- [ ] Owner + due date set for new action items.
- [ ] No unresolved ambiguity in ownership.
