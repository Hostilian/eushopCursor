# Readiness Triage Decision Aid

Use this aid for rapid triage when multiple blockers compete for attention.

## Priority rules

1. Any P0 or security risk with unknown impact = highest priority.
2. Gate blockers due within 72h outrank improvement tasks.
3. Evidence gaps for scheduled leadership decisions outrank optimization work.

## Triage matrix

| Urgency | Impact | Action |
| --- | --- | --- |
| High | High | Immediate owner assignment + same-day mitigation |
| High | Low | Time-box fix; defer if no gate impact |
| Low | High | Plan in current week with explicit owner |
| Low | Low | Backlog or bundle into next review |

## Quick triage checklist

- [ ] Severity assigned.
- [ ] Owner assigned.
- [ ] Due date set.
- [ ] Evidence requirement identified.
- [ ] Risk register update needed? (Y/N)

## Escalation trigger

Escalate to program manager if:

- No owner is available within 2 hours for high/high items.
- Two or more high/high blockers are open simultaneously.
- Any blocker threatens go/no-go evidence completeness.
