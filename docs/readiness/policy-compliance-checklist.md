# Readiness Policy Compliance Checklist

Use this checklist to verify policy conformance before each gate review.

## Governance controls

- [ ] Decision owners are explicitly named.
- [ ] Decision records include rationale and timestamp.
- [ ] Gate outcomes are logged in primary artifacts.

## Quality controls

- [ ] Required command checks completed for current scope.
- [ ] Critical journey checks are current.
- [ ] Incident drill evidence is current for planned period.

## Security/compliance controls

- [ ] Security/privacy checklist updated and reviewed.
- [ ] Exceptions include owner, approver, expiry, mitigation.
- [ ] No unreviewed high/critical vulnerability gaps.

## Evidence controls

- [ ] Evidence log has current cycle entries.
- [ ] Dashboard and weekly review statuses match source artifacts.
- [ ] KPI updates are traceable to source data.

## Operational controls

- [ ] Rollback path and owner are validated.
- [ ] On-call escalation matrix is up to date.
- [ ] Freeze/hotfix protocols are followed when applicable.
