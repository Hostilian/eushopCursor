# Ten-Minute Readiness Update Routine

Use this routine before daily standup or weekly review.

## Minute 0-2: Gates

- Open `go-no-go-decision.md`.
- Update gate statuses if changed.
- Capture new evidence dates.

## Minute 2-4: KPIs

- Open `readiness-scorecard.md`.
- Update any fresh KPI values/trends.
- Flag missing metrics as `data_missing`.

## Minute 4-6: Risks and blockers

- Open `risk-register.md` and `owner-action-register.md`.
- Promote top risks/blockers into dashboard.
- Confirm owner and due date exist.

## Minute 6-8: Dashboard refresh

- Update `command-center-dashboard.md`.
- Refresh date, cycle day, overall status.

## Minute 8-10: Log and escalate

- Add entry to `evidence-log.md`.
- If major shift, add decision-log entry in `30-day-master.md`.
- Notify owners of newly escalated items.
